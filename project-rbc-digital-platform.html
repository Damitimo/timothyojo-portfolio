From Fragmented to Unified: Building a Dynamic Insurance Platform That Reduced Application Processing Time by 50%

The Challenge

RBC Insurance offered three distinct products‚ÄîCritical Illness, Life, and Disability insurance‚Äîeach with unique eligibility criteria, workflows, and application processes. Field Sales Advisors struggled with a fragmented ecosystem, navigating four separate e-application platforms with inconsistent formats, validation processes, and user interfaces.

Key pain points identified:

  • Advisors couldn't copy-paste from Salesforce due to format requirements, forcing manual re-entry
  • Joint policies required duplicate data entry across platforms
  • Each product had different eligibility rules (citizenship, age limits, health conditions)
  • Inconsistent platform setups meant relearning processes for each product
  • This complexity drove advisors to choose competitors with more streamlined solutions

Business Impact: High advisor frustration, compliance errors, and competitive disadvantage with brokerage advisors preferring competitor platforms.

Research & Discovery

Research Objectives:
I led comprehensive user research with advisors and clients to understand the complete e-application experience and quantify inefficiencies.

Methodology:

  • Drop-off analysis to identify abandonment patterns
  • Time analysis measuring duration on each application section
  • Error pattern identification and impact on completion rates
  • User interaction analysis with help features and validation
  • A/B testing to evaluate solution iterations

Key Findings:
Through data analysis and advisor interviews, we identified core problems:

  • Complex, lengthy forms - Average 2-hour workflow across multiple platforms
  • Inadequate error handling - Unclear validation feedback causing rework
  • Insufficient progress tracking - No completion indicators across platforms
  • Privacy concerns - Client discomfort with sensitive information disclosure
  • Inconsistent UI/UX - Four different platforms with varying experiences
  • Manual data entry - Duplicate entry for joint policies and across systems

Quantified Impact:

  • Advisors spent 2 hours per application on average
  • Duplicate data entry for joint policies added 30+ minutes
  • Platform switching caused an average 20% error rate
  • High dropout rates during sensitive information sections

Product Strategy & Solution

Vision:
Create a universal e-application platform where advisors can seamlessly handle all three insurance products through a single, intelligent system that dynamically adjusts based on client information and product selections.

Core Product Requirements:
I defined product requirements addressing each identified problem:

  • Consolidated platform - Single system replacing four separate platforms
  • Simplified workflows - Intelligent form pre-filling from existing data
  • Robust error handling - Real-time validation with clear, actionable feedback
  • Progress tracking - Visual indicators and completion status
  • Privacy-first design - Phased information collection with clear explanations
  • Consistent experience - Unified UI/UX across all products

Product Architecture & Technical Approach

Business Analysis:
I conducted comprehensive analysis to map eligibility criteria across all three insurance products, creating a detailed matrix documenting:

  • Age limits and citizenship requirements per product
  • Health conditions and occupation restrictions
  • Product dependencies and combinations
  • Common elements vs. unique product-specific requirements

Technical Requirements:
Partnered with engineering to define requirements for:

  • Dynamic rules engine - Real-time eligibility checking and product dependency handling
  • Modular architecture - Each product (CI, Life, Disability) as a configurable component
  • Conditional logic - Dynamic field visibility based on selections
  • API integrations - Seamless data flow between Salesforce and illustration software
  • Scalability - Flexible configuration to accommodate future products
  • Compliance validation - Automated checks across all regulatory requirements

Key Features Delivered

For Advisors:

  • Smart product selection - Real-time eligibility checking eliminates invalid options
  • Intelligent pre-filling - Automatic population from existing Salesforce data
  • Unified workflow - Single platform replacing four separate systems
  • Seamless integrations - Automatic data transfer from illustration software
  • Progress tracking - Clear visibility into application status

For Business:

  • Modular configurations - Easy product rule updates without system rebuilds
  • Compliance automation - Built-in regulatory validation across all workflows
  • Cross-functional tools - Business teams can update rules without engineering support
