The Challenge

RBC's insurance illustration software served 2,500 advisors but had a Windows 97-era interface and fully offline architecture. This created competitive disadvantage against modernized competitors like IA and Manulife, while operational constraints prevented advisors from collaborating on complex illustrations, accessing work across devices, or recovering from device failures.

Through advisor interviews and usability testing, I identified critical pain points: inability to collaborate on complex cases, work trapped on individual devices, and perception that the outdated UI signaled less reliable tools compared to competitors. Advisors with remote work needs were particularly constrained by offline-only architecture.

Strategic Decision Framework

I developed a decision framework evaluating three strategic approaches for the Novinsoft collaboration. The analysis examined trade-offs across security risk, compliance requirements, ongoing maintenance, design consistency, future transferability, IP protection, sustainability, implementation speed, and scalability.

After analyzing the options, I recommended "Novinsoft Components with RIG Styling" to business leadership. This approach balanced development speed with security by sharing only visual design specifications rather than proprietary code, while maintaining good design consistency and low compliance overhead. The recommendation prioritized fast implementation and IP protection over perfect design consistency, accepting that minor component behavior differences were an acceptable trade-off for reduced security risk and faster time-to-market.

The analysis covered critical factors: security risks from external code sharing, compliance requirements for legal review and procurement, maintenance complexity for ongoing updates, intellectual property protection, and scalability for future vendor relationships. Each option was rated across these dimensions to inform the optimal technical architecture decision.

Novinsoft RIG Collaboration Decision Matrix

Technical architecture

Based on the decision framework analysis, I recommended the "Novinsoft Components with RIG Styling" approach, which achieved the optimal balance of speed, security, and brand consistency. This solution allowed Novinsoft to leverage their existing component library while applying RBC's visual design through exported styling properties, all built on a modern cloud-based architecture.

The technical architecture transformed the system from offline-only to cloud-enabled, enabling critical new capabilities: cloud data storage for persistent illustrations, multi-device access for advisor flexibility, real-time collaboration between advisors and colleagues, automatic backup and recovery, and seamless integration with RBC's broader digital ecosystem.

Key technical decisions included: maintaining low security risk by sharing only visual design specifications (no proprietary code), ensuring standard vendor agreement compliance, enabling periodic styling updates through automated processes, achieving excellent design consistency through actual RIG components, implementing secure cloud storage architecture, and establishing a seamless integration path for future RBC system transfers.

This architecture provided fast implementation speed with low complexity while preserving RBC's intellectual property, maintaining full control over brand presentation, and delivering modern cloud-based functionality that matched competitor capabilities.

Technical Architecture: Component Library + RIG Styling Integration

Cloud transformation

The modernization project fundamentally transformed the illustration software from a fully offline desktop application to a cloud-enabled platform, addressing critical operational limitations that had constrained advisor productivity for years. This architectural shift represented one of the most significant technical improvements in the modernization.

Key cloud capabilities delivered include: persistent cloud storage allowing advisors to save and access illustrations from any device, real-time collaboration enabling multiple advisors to work on complex illustrations together, automatic backup and recovery eliminating the risk of lost work, cross-device synchronization for seamless workflow continuity, and integration readiness for future RBC digital ecosystem connections.

The business impact was immediate: advisors could now start an illustration in the office and complete it at a client site, senior advisors could collaborate with junior staff on complex cases, and the risk of losing critical client work due to device failure was eliminated. This cloud transformation brought RBC's illustration capabilities in line with modern competitor offerings.

Partnership strategy

The partnership strategy with Novinsoft was designed to balance technical efficiency with strategic control. Rather than building from scratch or outsourcing completely, I architected a hybrid approach that leveraged Novinsoft's proven component library while maintaining RBC's design authority and intellectual property protection.

This approach delivered multiple strategic benefits: accelerated time-to-market by leveraging existing components, reduced development risk through proven technical foundation, maintained brand consistency through RIG design system integration, preserved RBC's IP by limiting code sharing to visual specifications, and established a scalable framework for future vendor collaborations.

Design system integration

I architected the technical integration between Novinsoft's component library and RBC's RIG design system, establishing a streamlined process for maintaining brand consistency without compromising development velocity. The solution involved exporting RIG design tokens, component specifications, and styling properties through our existing Storybook and Artifactory infrastructure.

The technical implementation included: automated export of design tokens (colors, typography, spacing) from RIG to Novinsoft's development environment, component specification documentation through Storybook for consistent implementation, CSS/SCSS theme files distributed via Artifactory for seamless integration, and automated testing to ensure visual consistency across both systems.

This approach achieved excellent design consistency while maintaining fast implementation speed and low complexity for ongoing maintenance, scoring as the optimal solution in our decision matrix analysis.

Impact

The modernized insurance illustration software delivered transformative business impact by addressing both user experience and operational efficiency challenges. The cloud transformation eliminated the offline limitations that had constrained advisor productivity, while the modern interface brought RBC's tools to competitive parity with industry leaders like IA and Manulife.

Key business outcomes included: enhanced advisor productivity through multi-device access and cloud storage, improved collaboration capabilities enabling team-based client service, reduced risk of lost work through automatic backup and recovery, accelerated client engagement through anywhere-access to illustrations, and strengthened competitive positioning with modern, professional-looking tools.

The technical architecture decisions ensured long-term sustainability: maintained brand consistency through RIG design system integration, preserved intellectual property through strategic vendor partnership, established scalable framework for future enhancements, and delivered modern development practices with pre-built components for faster iteration cycles.
